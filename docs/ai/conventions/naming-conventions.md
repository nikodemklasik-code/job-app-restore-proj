# Naming Conventions

## Library names

Use:
- `prompt-core`
- `interview-engine`
- `coach-engine`
- `persona-kit`
- `report-engine`
- `session-engine`

## File suffixes

Prefer:
- `*.prompt.ts`
- `*.schema.ts`
- `*.service.ts`
- `*.types.ts`
- `*.mapper.ts`
- `*.orchestrator.ts`

## Avoid

Avoid names such as:
- `final.ts`
- `newPrompt.ts`
- `helper.ts`
- `utils.ts`
- `v2.ts`
- `reallyFinal.ts`

## README rule

Each domain library must have its own `README.md`.
