# Squad Workboard

## Goal

Structure the team and enforce movement from documentation state into implementation state.

This workboard is binding for squad execution.

It is not a brainstorming board.
It is an implementation-phase execution board.

---

## Team Structure

- **Agent 1** = system foundations
- **Agent 2** = intelligence modules
- **Agent 3** = practice modules + settings/community
- **QC** = final gate
- **Product Owner** = final product meaning and priority owner

---

## Global Execution Rules

### Rule 1
Each agent implements only owned scope unless cross-scope integration is required.

### Rule 2
No agent may substitute more documentation for repository implementation.

### Rule 3
Each phase must end with:
- implementation output
- delivery report
- QC validation
- Product Owner signoff where required

### Rule 4
Nothing is considered done because it is well described.
It is done only when it is implemented in code, wired, and QC-approved.

---

## Required Delivery Format For Every Agent

Every phase delivery must include:

### Scope Implemented
### Files Changed
### Routes / APIs / Schemas / Components Changed
### Tests Added Or Updated
### Integration Notes
### Ready For QC: Yes / No
### Blockers

## Enforcement Rule

If a phase returns only:
- summaries
- plans
- design notes
- pseudo-completion
- or non-wired partial changes

then the phase is not complete and must return to implementation.

---

## Final Rule
Nothing is treated as done until:
- the owning agent completes implementation in repo
- QC approves it
- Product Owner signs off on user-facing meaning where relevant
